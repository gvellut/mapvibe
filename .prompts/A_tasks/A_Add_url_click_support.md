Make it so a dataLayers in the customUi section of a config can declare to be clickable. In that case, each feature is assumed to have a `url` field (next to possibly `title` or `description`). When clicking on the feature on the map, a new tab is open at the `url` location (different for each feature).
In that case, the layer still should have `"interactive": true`, but should have a field `openUrl` to `true`. Without the `openUrl`, it is the same behaviour as before (no change). If the feautre deos not have a `url` field, a log in the console should be made but no error otherwise (like a popup).
